## 工作流划分

### 工作流 1：表模型基础功能（用例 1-6）

**前置条件**：单节点 IoTDB，创建多个数据库和表，写入数据。

**节点链**：
1. `iotdb_deploy` — 部署 IoTDB
2. `iotdb_start` — 启动 IoTDB
3. `iotdb_cli`(table) — 建库建表写数据（db_g_0.t1, test1.t1/t2, empty_db, viewdb + view）
4. `iotdb_cli` x6 — 分别执行 6 条测试 SQL
5. 人工检查执行结果中的 stdout（当前 assert 节点无法直接引用上游 iotdb_cli 的 stdout）

**已实现**：`data/workflows/disk-usage-table-model-basic.workflow.json`

### 工作流 2：权限控制验证（用例 7-9）

**前置条件**：IoTDB 运行，创建普通用户。

**难点**：
- 需要用不同用户登录执行 SQL（`iotdb_cli` 的 username/password 参数支持）
- 需要验证 SQL 执行失败（预期 exit_status != 0）
- 用例 8 需要开启三权分立配置
- 用例 9 是树模型，需切换 sql_dialect

### 工作流 3：统计准确性验证（用例 10-11）

**前置条件**：IoTDB 运行，写入定量数据。

**难点**：
- 需要用 `shell` 节点执行 `du` / `ls` 获取 TsFile 实际大小
- 需要比对 SQL 查询结果和文件系统大小，计算误差
- 用例 10 关闭合并、用例 11 开启合并，需不同 iotdb_config

### 工作流 4：树模型查询（用例 12-16）

**前置条件**：IoTDB 运行（树模型），创建存储组，写入数据。

**节点链**：与工作流 1 类似，SQL 使用 `SHOW DISK_USAGE FROM root.xxx.**` 语法，`sql_dialect` 设为 `tree`。

### 工作流 5：分布式集群场景（用例 17-21）

**前置条件**：IoTDB 分布式集群（多 DataNode），特殊配置（TTL/多磁盘）。

**难点**：
- 需要 `iotdb_cluster_deploy` 部署多节点集群
- TTL 验证需 `wait` 节点等待 5 分钟
- 多磁盘验证需配置 `dn_data_dirs` 并对比磁盘实际大小

### 工作流 6：多级存储（用例 22-25）

**状态**：用例不完整（无测试步骤和预期结果），暂不可转换。

## 能力缺口

| 需求 | 现有能力 | 状态 |
|------|---------|------|
| SQL 结果列名验证 | iotdb_cli stdout + assert(custom) | 需组合 shell 解析 |
| 结果行数验证 | stdout 包含 "Total line number" | 用 assert(log_contains) |
| 不同用户登录 | iotdb_cli 的 username/password | 已支持 |
| 预期执行失败 | condition 节点判断 exit_status | 需反向验证 |
| 文件大小对比 | shell 节点 + assert(custom) | 需组合计算 |
| 等待 TTL 清理 | wait 节点 | 已支持 |
| 表/树模型切换 | iotdb_cli 的 sql_dialect | 已支持 |
