# 首页重设计：思维导图式模块展示

## 背景

当前首页包含：
- Hero Section（顶部品牌展示区，渐变背景）—— 用户认为"太丑了"，需要移除
- Flow Map（能力关系图）—— 已有流程图结构，但用户希望改成更清晰的思维导图形式

用户期望：以思维导图形式展示模块关系，中心一个大卡片，分叉出去显示子模块。

## 设计决策

### 1. 布局方式
**垂直树形展开**
- 顶部中心大卡片（起点模块）
- 向下分叉展开三个主分支
- 工作流分支继续向下延伸到运行分析

### 2. 视觉风格
**扁平简约**
- 白色卡片 + 边框，无阴影、无渐变
- 统一的视觉语言，所有层级保持一致
- 中心卡片和分支卡片风格统一，仅通过尺寸和颜色区分层级

### 3. 连接线样式
**细线 + 小节点**
- 1.5px 灰色线条（#cbd5e1）
- 3px 白色小圆点节点，带灰色描边
- 分支线不使用彩色，保持整体简洁

### 4. 层级结构
**完整版，所有层级统一风格**
- 第一层：服务器管理（起点）
- 第二层：系统监控、IoTDB可视化、工作流管理
- 第三层：运行分析（仅从工作流延伸）
- 第四层：底部信息区（快速信息展示）

---

## 组件设计

### 中心卡片（服务器管理）
- 白色背景，边框 `1px solid #e2e8f0`
- 圆角 `10px`
- 内含图标（蓝色背景）、标题、描述
- 尺寸较大，强调"起点"地位

### 分支卡片（监控、IoTDB、工作流）
- 白色背景，边框颜色对应模块主题色
  - 监控：`#10b981`（绿色）
  - IoTDB：`#ef4444`（红色）
  - 工作流：`#f59e0b`（橙色）
- 圆角 `6px`
- 内含 eyebrow 标签、标题
- 尺寸较小，横向排列

### 子节点卡片（运行分析）
- 白色背景，边框 `#8b5cf6`（紫色）
- 圆角 `6px`
- 仅从工作流向下延伸
- 风格与分支卡片一致

### 连接线
- 使用 SVG 绘制
- 主干线：`1.5px solid #cbd5e1`
- 分支节点：3px 圆点，白色填充，灰色描边
- 子层级延伸线同样使用细线风格

### 底部信息区
- 白色背景，边框 `1px solid #e2e8f0`
- 圆角 `6px`
- 展示快速信息：起点模块、分支数量、技术栈等
- 使用 flex wrap 布局，标签-值交替排列

## 布局参数

```

中心卡片：
  width: 180px
  padding: 16px
  border-radius: 10px

分支卡片：
  width: 80px
  padding: 12px 10px
  border-radius: 6px
  横向 gap: 12px

连接线区：
  height: 32px（主干线）
  SVG 绘制树形分支

子节点卡片：
  width: 90px
  padding: 10px 12px
  margin-left: 30px（偏移对齐工作流分支）

底部信息区：
  max-width: 320px
  padding: 10px 14px
  gap: 8px 16px（flex wrap）
```
